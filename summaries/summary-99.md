# ECMAScript仕様輪読会 第99回

## String.raw (template, ...substitutions)

`String.raw`は、テンプレートリテラルの「生の文字列」を取得するための関数です。主にタグ付きテンプレートリテラルとして使用されます。

### 仕組み
この関数は、第1引数として渡されるオブジェクトの`raw`プロパティを参照します。通常のテンプレートリテラルでは、バックスラッシュ（`\`）はエスケープシーケンスとして解釈されますが、`String.raw`を経由すると、入力されたそのままの文字として扱われます。

### 主な用途
Windowsのファイルパスや、正規表現のパターンを記述する際に、バックスラッシュを二重に書く手間を省くことができます。

```javascript
// 通常のテンプレートリテラル
const path = `C:\new\table`; 
// 'C:
// ew	able' （\nや\tが改行やタブとして解釈される）

// String.raw を使用
const rawPath = String.raw`C:\new\table`;
// 'C:\new\table' （そのままの文字列として取得できる）
```

## String.prototype オブジェクトの性質

`String.prototype`は、すべての文字列インスタンスが継承するプロパティやメソッドを持つオブジェクトです。

### 内部スロットとエキゾチックオブジェクト
*   **[[StringData]]**: この内部スロットには、そのオブジェクトが表す実際の文字列値が格納されています。`String.prototype`自体の初期値は空文字列です。
*   **エキゾチックオブジェクト**: 文字列オブジェクトは「String Exotic Object」と呼ばれ、通常のオブジェクトとは異なる挙動（インデックスによる文字アクセスなど）を内部メソッドのレベルでカスタマイズして実現しています。

### プリミティブとオブジェクトの挙動
文字列のプリミティブ値に対してメソッドを呼び出すと、内部的にオブジェクトへと変換（ラップ）されます。

```javascript
const s1 = "abc";           // プリミティブ
const s2 = new String("abc"); // オブジェクト

console.log(typeof s1); // "string"
console.log(typeof s2); // "object"
```

## 文字アクセス系のメソッド

文字列内の特定の文字や文字コードを取得するためのメソッド群について、その違いが議論されました。

### String.prototype.at (index)
ES2022で導入された比較的新しいメソッドです。負のインデックスを指定できるのが最大の特徴で、末尾からの位置指定が容易です。

*   インデックスが負の場合：`length + index` として扱われます。
*   範囲外の場合：`undefined` を返します。

```javascript
"あいうえお".at(-1); // "お"
"あいうえお".at(0);  // "あ"
```

### String.prototype.charAt (pos)
古くからあるメソッドで、指定した位置の文字を返します。

*   範囲外の場合：空文字列（`""`）を返します。
*   負のインデックス：サポートされておらず、空文字列を返します。

### String.prototype.charCodeAt (pos)
指定した位置の「コードユニット」（UTF-16の数値）を返します。サロゲートペア（絵文字や一部の漢字など）の場合、1文字分ではなく、半分（1つのコードユニット）の値しか返しません。

### String.prototype.codePointAt (pos)
Unicodeの「コードポイント」を返します。サロゲートペアを正しく解釈し、1つの完全な文字としての数値を返してくれるため、現代的なJavaScript開発ではこちらが推奨されます。

```javascript
const fish = "𩸽"; // サロゲートペアの文字

console.log(fish.charCodeAt(0));   // 55399 (前半のみ)
console.log(fish.codePointAt(0));  // 171581 (完全なコードポイント)

// 数値から文字に戻す場合
console.log(String.fromCodePoint(171581)); // "𩸽"
```

## String.prototype.concat (...args)

複数の文字列を連結するメソッドです。

1.  `this`の値を文字列に変換します。
2.  引数として渡された各値を順番に文字列に変換し、結合していきます。
3.  最終的な文字列の値を返します。

内部的にはコードユニットのシーケンスを単純につなげる処理であるため、サロゲートペアの断片同士をつなげて1文字を合成するといった操作も可能です。

```javascript
const parts = ["𩸽".charAt(0), "𩸽".charAt(1)];
console.log("".concat(...parts)); // "𩸽" （結合して元の文字に戻る）
```

## メソッドの汎用性（Genericness）

`String.prototype`にある多くのメソッドは「intentionally generic（意図的に汎用的）」に設計されています。つまり、`this`が文字列でなくても、`call`や`apply`を使って他のオブジェクトに対してメソッドを実行できます。

オブジェクトが渡された場合、まずそのオブジェクトを文字列に変換（`[object Object]`など）してから、その結果に対してメソッドの処理が行われます。

```javascript
const obj = { toString: () => "ABC" };
console.log(String.prototype.charAt.call(obj, 0)); // "A"
```

### 仕様書の記述漏れに関する指摘
勉強会の中では、`String.prototype.at`などの比較的新しい仕様において、他のメソッドには存在する「このメソッドは呼び出されると以下のステップを実行する」という定型文が欠落している箇所があることが発見されました。これは実動作には影響しませんが、仕様書の記述の一貫性に関する興味深い発見として議論されました。
